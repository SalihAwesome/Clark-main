01 The Problem

The web still makes you do the clicking.

Most "web work" isn't hard — it's mechanical. AI chat can describe a task; it still can't open the page and finish it.

Who's affected

Students and researchers pulling facts from many pages, professionals filing recurring paperwork, and anyone navigating dense, multi-step e-service portals that punish a single mistake.

Why it matters

Every manual, repetitive browser session is time not spent on the task the person actually cares about — and friction in bureaucratic workflows is friction in access to services.

Current pain points

Constant context-switching, unfamiliar UI patterns, logins/OTPs/CAPTCHAs that block naive automation, and no memory of what was typed in five minutes ago on a different site.

The gap

Chat can talk about the renewal form. It has never once opened it, read the fields, and filled them in for you.

02
02 Current Solutions

Chatbots talk. Scripts break. Neither finishes the job.

Plain chatbots

Limitation: no hands. They can explain a process in detail and still leave the actual clicking to you.

RPA / macro tools

Limitation: hard-coded, per-site scripts. One redesign and the whole automation silently breaks.

Narrow browser copilots

Limitation: reason about a single visible tab with no real multi-step planning, and no safe handling of logins or payments.

The opportunity

A general-purpose agent that reasons about any page it lands on — no site-specific script to write or maintain — and treats anything sensitive as a hard stop for a human.

03
03 Our Solution

One general-purpose agent. Any website.

Clark pairs a real, visible browser with an LLM reasoning loop (think → act → observe → repeat), so it operates sites it has never seen — not a fixed list of ones it was scripted for.

No fixed registry

Nothing to hard-code per site. The agent reasons about whatever page is actually in front of it.

Grounded clicking

A numbered Set-of-Marks overlay lets it click "box #14" instead of a selector a redesign can break.

Human-in-the-loop

Structurally unable to type a password, OTP, or full card number itself — by design, not policy.

Grounded answers

The final answer is re-derived from what it actually clicked and typed, not from what the model recalls.

06
06 AI Architecture

Think → Act → Observe. Repeat.

Pipeline

One JSON object per turn — a tool call or a final answer — parsed with a brace-matching fallback that tolerates stray text.

Models

Fireworks AI (DeepSeek-class, primary, AMD-accelerated) + Google Gemma 4 (multimodal, automatic fallback).

Agent

A single orchestrator (AgentRun) dispatches ~20 tools against a live, per-session browser worker.

Memory

Local "My Info" profile (incl. ID-photo OCR), saved credentials, saved payment card, full replayable audit trail.

Tool usage

Search, Set-of-Marks click/fill, smart date & text filling, login, submit, document generation.

Decision flow

Near the step budget, the loop forces a wrap-up; the final answer is re-derived from actual tool-call arguments.

07
07 AMD Technologies

Every primary inference call runs on AMD Instinct.

AMD Services Used

Clark's primary LLM path — every reasoning step, tool choice, and vision/OCR call through Fireworks AI — runs on AMD Instinct™ (MI300X / MI325X / MI355X-class) GPUs via Fireworks' FireAttention inference stack and the open ROCm™ ecosystem.

Why AMD

High-memory accelerators give headroom for DeepSeek-class models at the longer contexts an agentic loop needs, and ROCm's open stack keeps the inference layer vendor-neutral.

Performance Benefits

Per Fireworks' published FireAttention V3 benchmarks on AMD Instinct MI300 GPUs: up to 1.4×–1.8× throughput vs. comparable inference stacks — the same serving path behind Clark's step loop.

08 Innovation

What makes Clark hard to copy.

Technical innovations

Set-of-Marks grounded clicking, grounded-final-answer re-derivation from real tool calls, stealth-aware persistent browser sessions, and a per-provider circuit breaker with fast-fail timeouts.

Competitive advantage

No site-specific scripts to write or maintain. A transparent, replayable step trace. Safety that's structural — the model literally cannot type a credential itself.
